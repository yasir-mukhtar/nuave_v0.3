## Stale Audit Reaper

**Problem:** Audits stuck in `"running"` status when Vercel kills the serverless function before background processing completes. These ghost audits cause brands to show "No audits found" on the dashboard.

**Fix** (`src/app/api/audit/[id]/status/route.ts`):
- During status polling, if an audit has been `"running"` for >10 minutes, auto-mark as `"failed"`
- User sees error screen with "Coba lagi" (try again) button — already handled by running page
- Zero infrastructure cost — cleanup happens lazily during normal polling

**Database cleanup SQL** (run manually in Supabase SQL editor):
```sql
DELETE FROM recommendations WHERE audit_id IN (SELECT id FROM audits WHERE status IN ('running', 'failed', 'pending'));
DELETE FROM audit_results WHERE audit_id IN (SELECT id FROM audits WHERE status IN ('running', 'failed', 'pending'));
DELETE FROM audits WHERE status IN ('running', 'failed', 'pending');
```
